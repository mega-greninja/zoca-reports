7.The "what can the bot actually drive today" summary

🟢 Open

Square · Mindbody · Acuity

Full end-to-end: service search → slot → book → deposit → COF → cancel/reschedule → auto no-show charge. All via API in <2s. Thin middleware.

🟡 Closed-but-bookable

GlossGenius · Phorest

Book via API, but money (deposit + COF) always through SMS → our Stripe vault. Cancel/reschedule works. Salon-facing dashboard required.

🔴 Fully closed

Vagaro · Booksy · Fresha

Browser automation for booking (10–25s), our Stripe for all money. Cancel/reschedule slow. Multi-service + couples effectively unsupported.

8.Features worth singling out for product convos

  • Couple / 2-resource bookings — clean on Mindbody & Acuity; cart workaround on Square; browser-or-skip everywhere else.
  • Fixed vs % deposits — universal via our Stripe. Question is where the policy lives: in the platform config or in ours. Recommend ours.
  • Auto no-show charging — only open platforms do it natively. Closed platforms need dashboard + human click. Frame value-prop accordingly.
  • Class bookings — Mindbody-native; everyone else weak. Only matters if we go after yoga/fitness.
  • Double-SMS problem — platform confirmation + our deposit SMS. Send ours first, keep it short, or use Phorest's 10-min delay window.
  • Refund flow — if deposit is in our Stripe, salon can't refund from their UI. Needs to be in the Zoca dashboard.
