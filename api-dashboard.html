Service Variations & Add-ons (Modifiers)3▾
GET/v2/catalog/object/{ITEM_ID}?include_related_objects=trueGet service with ALL variations▶
Query params
include_related_objects=true
Each service (ITEM) has a variations[] array of ITEM_VARIATION objects — each variation is a separate bookable duration/price (e.g. 30/60/90 min, short/long hair). Booking uses service_variation_id, never item_id.
GET/v2/catalog/list?types=MODIFIER_LISTList modifier groups (customizations)▶
Square uses MODIFIER_LIST + MODIFIER to model customizations & add-ons (e.g. "+ Deep Conditioning $15"). Attached to items via modifier_list_info.
GET/v2/catalog/list?types=MODIFIERList individual modifiers / add-ons▶
Returns each modifier with price_money. For bookings, add-ons are modeled as extra appointment_segments (not modifiers on one segment) — modifiers apply at Order/Payment level.

Services / Session Types2▾
GET/site/sessiontypesBookable service types▶
Query params
programIDs=1
onlineOnly=true
limit=100
Each SessionType is a bookable service. "Variations" don't exist as a concept — each duration/variant is its own SessionType. Response includes DefaultTimeLength, DefaultTimeBetweenAppointments, Price.

No body. Filters: scheduleType (Appointment/Class/Enrollment).
